========================================
HOW TO RUN NEUROSTACK PROXY
========================================

You need TWO terminals open at the same time:

TERMINAL 1 (Server):
--------------------
1. Open CMD or PowerShell
2. Navigate to project:
   cd D:\Clients\neuro_router

3. Activate conda:
   conda activate mcc-project

4. (First run only) initialize the database:
   python -m app.database

5. Start the server:
   python run.py

6. You should see:
   INFO: Uvicorn running on http://0.0.0.0:7860
   INFO: Application startup complete.

7. KEEP THIS TERMINAL OPEN! Don't close it.


TERMINAL 2 (Client/Testing):
-----------------------------
1. Open a NEW CMD or PowerShell window
2. Navigate to project:
   cd D:\Clients\neuro_router

3. Activate conda:
   conda activate mcc-project

4. Run quick test:
   python quick_test.py

   OR run the client example:
   python client_example.py


TROUBLESHOOTING:
----------------
Q: Nothing happens when I run client_example.py?
A: Make sure Terminal 1 is running the server!

Q: I get "Connection refused" error?
A: Server is not running. Start it in Terminal 1.

Q: Server starts but no response from Groq?
A: Check your Groq API key in .env file is valid.

Q: How do I know server is working?
A: Visit http://localhost:7860 in your browser.
   You should see JSON response with service info.


TESTING IN BROWSER:
-------------------
While server is running, open browser:
http://localhost:7860

You should see:
{
  "service": "NeuroStack OpenAI Proxy",
  "status": "online",
  "endpoints": {...}
}


YOUR API ENDPOINTS:
-------------------
Base URL: http://localhost:7860/v1

Available:
- POST /v1/chat/completions
- POST /v1/completions  
- GET  /v1/models

Auth management (outside /v1):
- POST /auth/register
- POST /auth/login
- POST /api-keys
- GET  /api-keys
- DELETE /api-keys/{id}

Workflow to get an API key:
1) Register: POST /auth/register (email, password)
2) Login: POST /auth/login to get JWT
3) Create key: POST /api-keys with JWT -> raw key returned once
4) Use API key for /v1/* calls: Authorization: Bearer neurostack_xxxxxxxxxxxxx


TEST WITH CURL (while server running):
---------------------------------------
curl -X POST http://localhost:7860/v1/chat/completions ^
  -H "Authorization: Bearer YOUR_GENERATED_API_KEY" ^
  -H "Content-Type: application/json" ^
  -d "{\"model\":\"gpt-4\",\"messages\":[{\"role\":\"user\",\"content\":\"Hi\"}],\"max_tokens\":20}"


CHECKING GROQ DASHBOARD:
-------------------------
1. Make sure server is running (Terminal 1)
2. Make request from Terminal 2
3. Check Groq dashboard: https://console.groq.com
4. You should see API calls appear there
   (May take a minute to show up)


NEED HELP?
----------
1. Make sure .env file has valid GROQ_API_KEY
2. Check both terminals are in correct conda env
3. Ensure port 7860 is not used by another app
4. Try restarting the server
